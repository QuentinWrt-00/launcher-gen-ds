# ROLE-BREAKPOINTS.md
> Apply BASE.md + these rules. Use when adding responsive behavior to components.

---

## Breakpoint tokens

Breakpoints are defined in Figma and exported as CSS custom properties.
They are available in `_tokens.css` (do not edit this file directly).

```css
--breakpoints-mobile:  768px;
--breakpoints-tablet: 1024px;
--breakpoints-desktop: 1280px;
```

> ⚠️ CSS custom properties cannot be used inside `@media` queries — the browser
> resolves `var()` after layout, too late for media condition evaluation.
> The pipeline solves this by baking the token values directly into generated
> `@media` rules and `@theme` at build time. Run `npm run sync-tokens` after
> any Figma breakpoint change — everything updates automatically.

---

## Media query convention

This project uses a **mobile-first** approach: base styles target mobile,
overrides target larger screens.

```css
/* Mobile — base styles, no media query needed */
.component { font-size: 1rem; }

/* Tablet and above */
@media (min-width: 1024px) { /* --breakpoints-tablet */
  .component { font-size: 1.25rem; }
}

/* Desktop and above */
@media (min-width: 1280px) { /* --breakpoints-desktop */
  .component { font-size: 1.5rem; }
}
```

## In Tailwind v4

The pipeline automatically injects breakpoint values into `globals.css` from the
Figma tokens. **Do not edit this block manually** — it is overwritten on every
`npm run sync-tokens`.

```css
/* app/globals.css — generated automatically */
@theme inline {
  --breakpoint-tablet:  1024px;  /* ← value from Figma */
  --breakpoint-desktop: 1280px;  /* ← value from Figma */
}
```

Usage in components:

```tsx
// ✅ correct — uses Tailwind responsive prefix
<div className="text-sm tablet:text-base desktop:text-lg">...</div>

// ❌ incorrect — hardcoded breakpoint value
<div style={{ fontSize: window.innerWidth > 768 ? '1rem' : '0.875rem' }}>...</div>
```

> Note: the default Tailwind `sm:`, `md:`, `lg:`, `xl:` prefixes are not used —
> use `tablet:` and `desktop:` to stay aligned with the design system.

## Responsive overrides from Figma (auto-generated)

The pipeline detects Figma's "mobile" mode and automatically generates
`@media (max-width: 768px)` overrides in `_tokens.css`.
These cover typographic scale changes — do not duplicate them in components.

```css
/* Generated automatically — do not edit */
@media (max-width: 768px) {
  :root {
    --typography-headline-md-font-size: var(--typography-font-size-2xl);
    /* ... */
  }
}
```

- [ ] Components use `tablet:` / `desktop:` prefixes, not `md:` / `lg:`
- [ ] No breakpoint value hardcoded in component files
- [ ] If Figma breakpoints changed → run `npm run sync-tokens` (globals.css updates automatically)
