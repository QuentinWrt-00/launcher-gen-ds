# ROLE-FONTS.md
> Apply BASE.md + these rules. Use when setting up or auditing fonts on a project.

---

## 1. Declare fonts with @font-face

Place `.woff2` files in `public/fonts/`. One `@font-face` block per weight/style variant. Always declare `font-display: swap`.

```css
/* app/fonts.css — imported once in globals.css */

@font-face {
  font-family: "Brand Serif";
  src: url("/fonts/BrandSerif-Regular.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

@font-face {
  font-family: "Brand Serif";
  src: url("/fonts/BrandSerif-Light.woff2") format("woff2");
  font-weight: 300;
  font-style: normal;
  font-display: swap;
}

@font-face {
  font-family: "Brand Sans";
  src: url("/fonts/BrandSans-Regular.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

@font-face {
  font-family: "Brand Sans";
  src: url("/fonts/BrandSans-Light.woff2") format("woff2");
  font-weight: 300;
  font-style: normal;
  font-display: swap;
}
```

## 2. Register fonts in globals.css via @theme

Tailwind v4 reads `--font-*` custom properties from `@theme inline`.
These must reference the tokens from `_tokens.css` — never hardcode font names here.

```css
/* app/globals.css */
@import "tailwindcss";
@import "./fonts.css";      /* @font-face declarations */
@import "./_tokens.css";    /* generated — do not edit */

@theme inline {
  --font-sans:  var(--typography-font-family-secondary);  /* Apercu Pro, etc. */
  --font-serif: var(--typography-font-family-primary);    /* Brand Serif, etc. */
}
```

---

## 3. Font fallbacks

The pipeline automatically appends `system-ui, sans-serif` to every font family token.
No configuration needed — it works out of the box for any project.

```css
/* Generated automatically */
--typography-font-family-primary:   "Brand Font", system-ui, sans-serif;
--typography-font-family-secondary: "Body Font",  system-ui, sans-serif;
```

**If the brand font is a serif**, edit the `fontFamily` case in `scripts/tokens-to-css.js`:

```js
// Before (generic)
return `${quoted}, system-ui, sans-serif`;

// After (serif project)
return `${quoted}, Georgia, "Times New Roman", serif`;
```

Then run `npm run sync-tokens`.

---

## 4. Use fonts in components

Always use the `@utility` classes generated by the pipeline — never reference font tokens directly in components.

```tsx
// ✅ correct
<p className="typo-body-md">...</p>

// ❌ incorrect — bypass the design system
<p style={{ fontFamily: 'Brand Serif' }}>...</p>
<p className="font-serif text-base">...</p>
```

---

## Checklist before delivering

- [ ] `.woff2` files present in `public/fonts/`
- [ ] One `@font-face` block per weight variant, `font-display: swap` on each
- [ ] `FONT_FALLBACKS` filled in `tokens-to-css.js`
- [ ] `npm run sync-tokens` run after any change
- [ ] No font name hardcoded in components or config files
